HTML

  • head element- contains information about the webpage; not visible to the user
  • body element- represents the visible content shown to the user
  • title element- defines the title for a webpage; required; important for Search Engine Optimization (SEO)
  • style element- defines CSS styles associated with the website
  • base element- defines the base URL for a webpage
  • link element- connects an external resource to the HTML document
  • meta element- defines metadata such as the character set, description, keywords, author, and viewport
  • charset- defined in the meta element; e.g., UTF-8, which allows us to use most characters and symbols on the page

CSS

  • margin- indicates how much space we want around the outside of an element
  • padding- indicates how much space we want around the content inside of an element

Git

GitHub Workflow:

  • Create a branch
  • Make changes
  • Create a pull request
  • Address review comments
  • Marge pull request
  • Delete branch

Git Commands

  • "git checkout -b feature/name" - creates feature branch
  • "git add -A" - adds code to working branch
  • "git commit -m "name"" - commits code to working branch and includes commit message
  • "git pull origin main" - ensures local branch is in sync with base branch in GitHub
  • "git status" - lets you know which branch you are currently working on locally
  • "git push origin feature/name" - pushes changes to the remote branch

JavaScript

  • variable- a named container that allows us to store data in our code
  • control flow- the order in which a computer executes code in a script
  • function- set of instructions that tells a computer how to perform a certain task
  • conditional statements- (and loops) control the order in which code is executed and whether it is executed at all
